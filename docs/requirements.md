========================================
要件定義書
OsiCal - 推し活スケジュール管理アプリ
========================================

1. 概要（What）
----------------------------
OsiCal は、推し活に関するイベント（ライブ、配信、応募期間、特典会など）を
1つのカレンダーで管理するためのアプリケーションである。
推し名やイベント種別（タグ）を自由に追加し、予定をわかりやすく整理することを目的とする。

2. 背景・目的（Why）
----------------------------
推し活ではライブ・配信・応募締切など多くの予定が発生し、
その情報が個人のメモや複数のアプリに分散して管理が複雑になるという課題がある。
OsiCal はそれらの予定を一元管理できるようにし、
スケジュール管理の負担を軽減することを目的とする。

3. 成功基準（Definition of Done）
----------------------------
以下の最低限の機能が動作すれば完了とする：

・推し名の登録（既存選択＋新規追加）
・イベント登録（既存タグ選択＋新規タグ追加）
・イベント一覧をカレンダーで表示
・イベントの詳細確認
・イベントの編集、削除
・推し別のイベント一覧表示

4. スコープ（Scope）
----------------------------

【対象範囲（やること）】
・推し名管理（名前のみ管理）
・イベント登録、編集、削除
・タグ（イベント種別）管理
・カレンダー表示（FullCalendar）
・セッション管理（表示月保持など）
・推し別イベント一覧

【非対象（やらないこと）】
・ログイン機能
・通知機能
・画像アップロード
・当落管理
・多ユーザー対応
※ 上記は今後の拡張とする

5. 機能一覧（詳細は別資料）
----------------------------
・推し名管理機能（追加・選択）
・タグ管理機能（追加・選択）
・イベント登録
・イベント編集
・イベント削除
・イベント詳細表示
・カレンダー表示機能
・セッション保持機能

6. 非機能要件
----------------------------
・個人利用を前提としたローカル実行
・レスポンス 1 秒以内
・PHP（MVC構造）で開発
・拡張性を考慮し、DB構造はシンプルに保つ

7. データ要件（概要）
----------------------------

【推し（oshis）】
id
oshi_name

【タグ（tags）】
id
tag_name

【イベント（events）】
id
oshi_id（FK）
title
日付・時間・会場などの任意項目
memo
created_at

【イベントタグ（event_tags）】
event_id
tag_id

（詳細仕様・SQL は DB設計書側で定義）

----------------------------------------
※ 詳細仕様は以下の別資料で管理する
- DB設計書（詳細カラム、SQL含む）
- ER図
- 画面遷移図
- 画面設計（ワイヤーフレーム）
- 技術スタック
- 環境構築手順書
----------------------------------------
